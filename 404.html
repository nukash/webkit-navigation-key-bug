history.replaceState() corrupts NavigationHistoryEntry.key

WebKit Bugzilla – minimal step-by-step reproduction

1 Navigate to Page A
history.pushState({}, '', '/page-a')
2 Navigate to Page B
history.pushState({}, '', '/page-b')
3 Go back to Page A
history.back()
4 Call replaceState on Page A
history.replaceState({ updated: true }, '')
What this tests:
Per the spec, NavigationHistoryEntry.key identifies a slot in the history list. Calling replaceState() updates the entry's state, but the slot is the same — so the key must not change.

Impact: Vue Router, React Router, and other SPA frameworks call replaceState() before every navigation. This causes all key values to become unstable, breaking navigation.traverseTo(savedKey) and any code that relies on key stability for state management.
